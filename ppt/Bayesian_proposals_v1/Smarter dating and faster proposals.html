Rate parameterisations

  • In Bayesian inference, the way a model is parameterised can have profound effects on the efficiency of MCMC

  • Re-parameterisations can also change the meaning of the model itself

  • Clock rates r⃗ r→ are to be re-parameterised:

Why categories?

  • The key advantage of the cat parameterisation: there is no need to estimate the prior density p(p(⃗ R→||σσ)) because it is uniform

  • And this improves mixing efficiency in MCMC

  • But since cat is discrete, the Constant Distance operators by Zhang and Drummond 2020 do not apply

  • The Constant Distance operators can improve mixing efficiency by an order of magnitude when sequences are long

If only there was a way to get the best of both worlds...

Why quantiles?

  • There is no need to estimate the prior density p(p(⃗ R→||σσ)) because it is uniform ✅

  • As the domain of ⃗ R→ is continuous, we can use the Constant Distance operators ✅

  • Using a linear piecewise approximation → don't need to calculate the expensive cdf FF or i-cdf F−1F−1 too often ✅

Benchmarking

Two independent MCMC chains are run until:
  1. Each clade posterior probability has a difference between the two chains of <0.05<0.05
  2. The effective sample size (ESS) of each parameter is >200>200
Benchmarking each setting 10×10× using 4 datasets:
  1. The spider dataset by Richart et al. 2015 (6 taxa, 14 kb)
  2. The caterpillar dataset by Kawahara et al. 2013 (70 taxa, 2 kb)
  3. The songbird dataset by Moyle et al. 2016 (106 taxa, 0.6 kb)
  4. Simulated data (40 taxa, 3 kb)
The 3 empirical datasets (and more) were compiled and partitioned by Lanfear (see BenchmarkAlignments GitHub repository)

Proposal kernels

In Bayesian MCMC, the probability of accepting proposal θ′θ′ from state θθ is:

α(θ′|θ)=min(1,α(θ′|θ)=min(1, p(θ′|D)p(θ|D)p(θ′|D)p(θ|D) q(θ|θ′)q(θ′|θ)q(θ|θ′)q(θ′|θ) ))

where q(x|y)q(x|y) is the proposal kernel

Under a random walk proposal kernel:

x←y+sx←y+sΣΣ

for some random variable ΣΣ (typically a Uniform distribution) and tunable parameter ss
22 / 42

A good proposal kernel

A proposal kernel q(x|y)q(x|y) should make step sizes that are:
  • Sufficiently large so that vast areas of parameter space are explored

  • But if the steps are too large and the changes are too bold, the proposal will always be rejected

Roberts et al. 1997 describe an optimal acceptance probability of 23.4%

Bactrian kernel

  • The Bactrian distribution places small probability on small steps and small probability on large steps

  • And high probability on medium steps

  • Yang et al. 2013 describe an optimal acceptance probability of 30-40%, depending on the value of mm

  • A Bactrian(m=0.95m=0.95) proposal kernel is recommended

  • We will compare Bactrian(m=0.95m=0.95) and Bactrian(m=0.98m=0.98) kernels with the standard Uniform kernel

  • This kernel will affect clock model parameters: clock standard deviation σσ, rate quantiles qq, and internal node heights tt

Narrow Exchange

The Narrow Exchange operator moves the B−DB−D branch onto the C−EC−E branch without changing any node heights

31 / 42

Can we do better?

Lakner et al. 2008 find that tree operators are best when they change either
  1. Tree topology

  2. Branch lengths
but not both. Otherwise the tree prior and likelihood may both be strongly affected.

What if we extended Narrow Exchange so that the genetic distances between nodes are conserved?

Maybe we can propose node heights (time) as well as topology 🤔

Narrow Exchange Rates (NER)

The Narrow Exchange Rate operator moves the B−DB−D branch onto the C−EC−E branch. The branch rates are recalculated so that genetic distances are conserved. The DD node height may also be proposed.
